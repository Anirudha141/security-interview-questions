Network Security

30 interview questions covering protocols, firewalls, IDS/IPS, and network attacks

← Back to Categories
Q1 Explain the OSI model and its security relevance. Easy +

OSI 7 Layers and Security:

  • Layer 7 (Application): Web attacks (XSS, SQLi), malware, WAF protection
  • Layer 6 (Presentation): SSL/TLS encryption, encoding attacks
  • Layer 5 (Session): Session hijacking, authentication
  • Layer 4 (Transport): TCP/UDP, port scanning, SYN floods, firewalls
  • Layer 3 (Network): IP spoofing, routing attacks, IPSec
  • Layer 2 (Data Link): ARP spoofing, MAC flooding, 802.1X
  • Layer 1 (Physical): Physical access, wiretapping, signal jamming

Security controls by layer:

  • L2: Port security, VLAN segmentation, 802.1X
  • L3/L4: Firewalls, ACLs, IPSec VPN
  • L7: WAF, IDS/IPS, DLP
Q2 What is the TCP three-way handshake and how can it be exploited? Medium +

TCP Three-Way Handshake:

  1. SYN: Client sends SYN packet with initial sequence number
  2. SYN-ACK: Server responds with SYN-ACK, acknowledges client's SYN
  3. ACK: Client sends ACK, connection established

Exploitation - SYN Flood Attack:

  • Attacker sends many SYN packets with spoofed source IPs
  • Server allocates resources for half-open connections
  • Server's connection table fills up
  • Legitimate connections denied

Other attacks:

  • TCP Reset: Send RST to terminate connections
  • Session hijacking: Predict sequence numbers
  • Port scanning: SYN scan, connect scan, FIN scan

Defenses: SYN cookies, rate limiting, firewall rules, IPS

Q3 Explain ARP poisoning and how to detect/prevent it. Medium +

ARP Poisoning exploits ARP protocol's lack of authentication to associate attacker's MAC with another IP (usually gateway).

Attack flow:

  1. Attacker sends gratuitous ARP replies
  2. Victim updates ARP cache with attacker's MAC
  3. Traffic intended for gateway goes to attacker
  4. Attacker forwards to gateway (MITM position)

Tools: arpspoof, Ettercap, Bettercap

Detection:

  • ARP cache monitoring (arpwatch)
  • Duplicate IP detection
  • IDS signatures
  • Check gateway MAC: arp -a

Prevention:

  • Static ARP entries (small networks)
  • Dynamic ARP Inspection (DAI)
  • 802.1X authentication
  • VPN for sensitive traffic
  • Network segmentation
Q4 What is the difference between IDS and IPS? Easy +

IDS (Intrusion Detection System):

  • Passive monitoring
  • Detects and alerts on suspicious activity
  • Does NOT block traffic
  • Typically placed on span/mirror port
  • No impact on network performance
  • May have false positives without blocking legitimate traffic

IPS (Intrusion Prevention System):

  • Active, inline deployment
  • Detects AND blocks malicious traffic
  • Sits in traffic path
  • Can impact network performance
  • False positives can block legitimate traffic
  • Requires careful tuning

Detection methods (both):

  • Signature-based: Known attack patterns
  • Anomaly-based: Deviation from baseline
  • Heuristic: Behavior analysis

Examples: Snort, Suricata, Zeek (IDS), Palo Alto, Cisco Firepower (IPS)

Q5 How does a firewall work and what types exist? Easy +

Firewall controls network traffic based on predetermined rules, allowing or blocking packets.

Types of firewalls:

  • Packet filtering: Examines headers (IP, port), stateless, fast but limited
  • Stateful inspection: Tracks connection state, more secure than packet filtering
  • Application layer (proxy): Deep packet inspection, understands protocols
  • Next-Gen Firewall (NGFW): IPS + app awareness + threat intelligence
  • WAF: Web Application Firewall, HTTP-specific protection

Common firewall rules:

  • Allow established connections
  • Block all inbound by default
  • Allow specific services (80, 443)
  • Log and block suspicious activity

Bypass techniques (for testing):

  • Tunneling (DNS, ICMP, HTTP)
  • Fragmentation
  • Non-standard ports
  • Protocol abuse
Q6 Explain VLAN hopping attacks and prevention. Hard +

VLAN Hopping allows attacker to access traffic on other VLANs, bypassing L2 segmentation.

Attack methods:

1. Switch Spoofing:

  • Attacker negotiates trunk link with switch
  • Uses DTP (Dynamic Trunking Protocol)
  • Gains access to all VLANs on trunk

2. Double Tagging:

  • Attacker sends frame with two 802.1Q tags
  • First tag matches native VLAN (stripped by first switch)
  • Second tag specifies target VLAN
  • One-way attack (can send but not receive replies)

Prevention:

  • Disable DTP: switchport nonegotiate
  • Set access ports explicitly: switchport mode access
  • Change native VLAN to unused VLAN
  • Tag native VLAN traffic: vlan dot1q tag native
  • Disable unused ports
Q7 How does DNS work and what are DNS attacks? Medium +

DNS Resolution Process:

  1. Client queries recursive resolver
  2. Resolver checks cache, if miss queries root server
  3. Root directs to TLD server (.com, .org)
  4. TLD directs to authoritative nameserver
  5. Authoritative returns IP address
  6. Resolver caches and returns to client

DNS Attacks:

  • DNS Spoofing/Cache Poisoning: Insert false records into cache
  • DNS Hijacking: Modify DNS settings or responses
  • DNS Tunneling: Exfiltrate data via DNS queries
  • DNS Amplification: DDoS using open resolvers
  • Domain Hijacking: Compromise registrar account
  • Subdomain Takeover: Claim abandoned DNS records

Defenses:

  • DNSSEC (cryptographic signing)
  • DNS over HTTPS/TLS (DoH/DoT)
  • Response Rate Limiting
  • Monitor DNS traffic patterns
Q8 What is a Man-in-the-Middle (MITM) attack? Medium +

MITM Attack positions attacker between two communicating parties to intercept/modify traffic.

Attack vectors:

  • ARP Spoofing: Redirect traffic at L2
  • DNS Spoofing: Redirect at name resolution
  • Rogue AP: Evil twin WiFi access point
  • BGP Hijacking: Route internet traffic through attacker
  • SSL Stripping: Downgrade HTTPS to HTTP
  • DHCP Spoofing: Provide malicious gateway/DNS

What attackers can do:

  • Capture credentials
  • Inject malicious content
  • Modify transactions
  • Steal session tokens

Prevention:

  • HTTPS with HSTS
  • Certificate pinning
  • VPN
  • Network security controls (DAI, DHCP snooping)
  • Verify certificate warnings
Q9 How do you perform network reconnaissance? Medium +

Passive Reconnaissance:

  • OSINT: WHOIS, DNS records, BGP info
  • Shodan/Censys: Internet-wide scan data
  • Certificate transparency: Discover subdomains
  • Job postings: Technology stack hints

Active Reconnaissance:

  • Host discovery: 
    nmap -sn 192.168.1.0/24
  • Port scanning: 
    nmap -sS -sV -p- target
  • Service enumeration: 
    nmap -sC -sV target
  • OS detection: 
    nmap -O target
  • Vulnerability scanning: Nessus, OpenVAS, Nuclei

Protocol-specific:

  • SNMP: snmpwalk
  • SMB: smbclient, enum4linux
  • LDAP: ldapsearch
Q10 Explain SSL/TLS and common vulnerabilities. Medium +

TLS Handshake (simplified):

  1. Client Hello: Supported cipher suites, random
  2. Server Hello: Selected cipher, certificate
  3. Key Exchange: Generate session keys
  4. Encrypted communication begins

Historical vulnerabilities:

  • POODLE: SSLv3 padding oracle attack
  • BEAST: CBC cipher attack in TLS 1.0
  • Heartbleed: OpenSSL memory disclosure
  • CRIME/BREACH: Compression-based attacks
  • DROWN: Cross-protocol attack using SSLv2
  • ROBOT: RSA padding oracle

Testing:

testssl.sh target.com
nmap --script ssl-enum-ciphers -p 443 target

Best practices:

  • TLS 1.2+ only (prefer 1.3)
  • Strong cipher suites (AEAD)
  • Valid certificates
  • HSTS enabled
Q11 What are the different types of port scans? Medium +

Nmap Scan Types:

  • TCP Connect (-sT): Full handshake, logged, reliable
  • SYN Scan (-sS): Half-open, stealthy, fast (default with root)
  • UDP Scan (-sU): Slow, unreliable responses
  • FIN Scan (-sF): Sends FIN, closed ports respond with RST
  • NULL Scan (-sN): No flags set
  • Xmas Scan (-sX): FIN+PSH+URG flags
  • ACK Scan (-sA): Determine firewall rules, not open ports
  • Idle Scan (-sI): Use zombie host, very stealthy

Port states:

  • Open: Service listening
  • Closed: No service, responds with RST
  • Filtered: Firewall blocking, no response
  • Open|Filtered: Cannot determine (UDP)

Examples:

nmap -sS -p 1-1000 target      # SYN scan top 1000
nmap -sU -p 53,161,500 target  # UDP specific ports
nmap -sA target                 # Firewall rule detection
Q12 How does NAT work and what are its security implications? Easy +

NAT (Network Address Translation) maps private IPs to public IPs, conserving IPv4 addresses.

Types:

  • Static NAT: 1:1 mapping (private to public)
  • Dynamic NAT: Pool of public IPs
  • PAT/NAPT: Many:1 using port numbers (most common)

Security benefits:

  • Hides internal network structure
  • Blocks unsolicited inbound connections
  • Creates implicit firewall effect

Security limitations:

  • NOT a security control (security through obscurity)
  • Port forwarding exposes internal services
  • Breaks IPSec AH, some applications
  • Doesn't protect outbound malicious traffic

NAT traversal for attackers:

  • Reverse shells (outbound connections)
  • NAT hole punching
  • Application-layer tunneling
Q13 What is network segmentation and why is it important? Easy +

Network Segmentation divides a network into smaller, isolated segments to limit lateral movement and contain breaches.

Methods:

  • VLANs: Logical separation at L2
  • Subnets: IP-based separation at L3
  • Firewalls: Control traffic between segments
  • Microsegmentation: Application-level isolation
  • DMZ: Separate zone for public-facing services

Benefits:

  • Limits blast radius of breaches
  • Protects sensitive systems (PCI, healthcare)
  • Improves monitoring and visibility
  • Reduces attack surface
  • Compliance requirements (PCI-DSS)

Common segments:

  • User workstations
  • Servers/databases
  • Management network
  • Guest WiFi
  • IoT devices
  • DMZ for public services
Q14 Explain DHCP attacks and defenses. Medium +

DHCP Attacks:

1. DHCP Starvation:

  • Flood server with DISCOVER messages using spoofed MACs
  • Exhaust IP address pool
  • Legitimate clients can't get addresses

2. Rogue DHCP Server:

  • Set up malicious DHCP server
  • Provide attacker-controlled gateway/DNS
  • Enables MITM attacks

Attack tools: Yersinia, DHCPig

Defenses:

  • DHCP Snooping: Track legitimate DHCP messages, create binding table
  • Trusted ports: Only allow DHCP offers from known server ports
  • Rate limiting: Limit DHCP messages per port
  • Port security: Limit MACs per port

Cisco example:

ip dhcp snooping
ip dhcp snooping vlan 10
interface Gi0/1
  ip dhcp snooping trust
Q15 What is BGP and how can it be attacked? Hard +

BGP (Border Gateway Protocol) is the routing protocol of the internet, exchanging routing information between Autonomous Systems (AS).

BGP Attacks:

  • BGP Hijacking:
    • Announce routes for IP space you don't own
    • Traffic rerouted through attacker's AS
    • Enables mass MITM, traffic interception
  • Prefix Hijacking: Announce more specific prefix (wins routing)
  • Route Leak: Incorrectly propagate routes
  • BGP DoS: Flood with route updates

Real incidents:

  • YouTube hijacked by Pakistan Telecom (2008)
  • Route leaks causing internet outages

Defenses:

  • RPKI (Resource Public Key Infrastructure)
  • BGP Route Origin Validation
  • IRR filtering
  • Monitoring (BGPStream, RIPE RIS)

Q17 What is a DDoS attack and how is it mitigated? Medium +

DDoS (Distributed Denial of Service) overwhelms target with traffic from multiple sources.

Attack types:

  • Volumetric: Flood bandwidth (UDP flood, amplification)
  • Protocol: Exhaust resources (SYN flood, Ping of Death)
  • Application layer: Target specific services (HTTP flood, Slowloris)

Amplification attacks:

  • DNS amplification (70x)
  • NTP amplification (556x)
  • Memcached amplification (50000x)

Mitigation strategies:

  • Scrubbing services: Cloudflare, Akamai, AWS Shield
  • Rate limiting: Limit requests per IP
  • Anycast: Distribute traffic globally
  • Blackholing: Drop traffic to affected IP (last resort)
  • CDN: Absorb traffic at edge
  • BCP38: Filter spoofed packets at ISP level
Q18 Explain VPN technologies and their security considerations. Medium +

VPN Types:

  • IPSec: Network layer, site-to-site or remote access, complex setup
  • SSL/TLS VPN: Application layer, easier deployment, works through NAT
  • WireGuard: Modern, fast, simpler codebase
  • OpenVPN: Open source, flexible, uses TLS

IPSec modes:

  • Transport: Encrypts payload only
  • Tunnel: Encrypts entire packet, new IP header

Security considerations:

  • Cipher suite configuration
  • Certificate vs PSK authentication
  • Split tunneling risks (traffic leakage)
  • DNS leak prevention
  • Kill switch functionality
  • VPN gateway as single point of failure

Testing: Check for IP/DNS leaks, proper encryption, certificate validation

Q19 What is 802.1X and how does it work? Medium +

802.1X is a port-based network access control standard that authenticates devices before granting network access.

Components:

  • Supplicant: Client requesting access
  • Authenticator: Switch/AP controlling port
  • Authentication Server: RADIUS server validating credentials

Process:

  1. Supplicant connects to port (blocked state)
  2. Authenticator requests identity
  3. Supplicant provides credentials (EAP)
  4. Authenticator forwards to RADIUS
  5. RADIUS validates, returns accept/reject
  6. Port authorized or remains blocked

EAP Methods:

  • EAP-TLS: Certificate-based (most secure)
  • PEAP: Password in TLS tunnel
  • EAP-TTLS: Similar to PEAP

Security benefits: Prevents unauthorized access, VLAN assignment, device posture checking

Q20 How do you secure a wireless network? Medium +

Wireless Security Evolution:

  • WEP: Broken, never use
  • WPA: TKIP, deprecated
  • WPA2: AES-CCMP, current standard
  • WPA3: SAE, strongest (use if supported)

Security measures:

  • Strong encryption: WPA3 or WPA2-Enterprise
  • Complex passphrase: 15+ characters for WPA2-PSK
  • 802.1X authentication: For enterprise
  • Disable WPS: PIN brute-force vulnerability
  • Client isolation: Prevent inter-client attacks
  • Rogue AP detection: WIDS/WIPS
  • Separate guest network
  • Hidden SSID: Minimal security benefit
  • MAC filtering: Easily bypassed, not a security control

Attacks to test: Deauth, evil twin, KRACK (WPA2), dictionary attack

Q21 What is SNMP and its security risks? Medium +

SNMP (Simple Network Management Protocol) is used to manage and monitor network devices.

Versions:

  • SNMPv1/v2c: Community strings (cleartext passwords)
  • SNMPv3: Authentication and encryption

Security risks:

  • Default community strings: public (read), private (write)
  • Information disclosure: System info, network topology, routing tables
  • Configuration modification: Write access can reconfigure devices
  • SNMP reflection/amplification: DDoS attacks

Enumeration:

snmpwalk -v2c -c public target
nmap -sU -p161 --script snmp-brute target
onesixtyone -c community.txt target

Hardening:

  • Use SNMPv3 with authentication/encryption
  • Change default community strings
  • Restrict SNMP access by IP (ACLs)
  • Disable SNMP if not needed
Q22 Explain IPv6 security considerations. Hard +

IPv6 Security Considerations:

New attack surfaces:

  • Neighbor Discovery (ND): Replaces ARP, similar spoofing attacks
  • Router Advertisement (RA) attacks: Rogue router injection
  • Extension headers: Can bypass security controls
  • Tunneling (6to4, Teredo): May bypass IPv4 security controls

Common issues:

  • IPv6 often enabled but not monitored
  • Firewalls may not filter IPv6
  • Dual-stack complexity
  • Large address space complicates scanning

Attacks:

  • SLAAC attack: Fake RA to become default gateway
  • DHCPv6 spoofing: Similar to IPv4
  • NDP exhaustion: Fill neighbor cache

Defenses:

  • RA Guard
  • DHCPv6 guard
  • SEND (Secure Neighbor Discovery)
  • Consistent security policies for v4/v6
Q23 What is SMB and common SMB vulnerabilities? Medium +

SMB (Server Message Block) is a file sharing protocol used extensively in Windows environments.

Notable vulnerabilities:

  • EternalBlue (MS17-010): Remote code execution, used by WannaCry
  • SMBGhost (CVE-2020-0796): SMBv3 compression RCE
  • SMB Signing disabled: Enables relay attacks
  • Null sessions: Anonymous enumeration
  • SMBv1: Multiple vulnerabilities, deprecated

Enumeration:

nmap -p445 --script smb-enum-shares target
smbclient -L //target -N
enum4linux -a target
crackmapexec smb target

Security measures:

  • Disable SMBv1
  • Enable SMB signing
  • Block port 445 at perimeter
  • Apply security patches
  • Restrict null sessions
  • Use network segmentation
Q24 How do you test network device security? Medium +

Network Device Testing Areas:

  • Default credentials: Admin/admin, cisco/cisco
  • Management interfaces: SSH, HTTPS, SNMP, Telnet
  • Firmware vulnerabilities: Check CVEs
  • Configuration review: Unused services, weak crypto
  • Access control: Management ACLs, RBAC

Testing approach:

  1. Discovery: Identify devices, versions
  2. Authentication testing: Default creds, brute force
  3. Service enumeration: SNMP, HTTP, SSH versions
  4. Vulnerability scanning: Nessus, Nexpose
  5. Configuration audit: Against CIS benchmarks

Common issues:

  • Unpatched firmware
  • Default or weak passwords
  • SNMP with default community
  • HTTP management (not HTTPS)
  • Telnet enabled
  • Verbose banners
Q25 What is network forensics? Medium +

Network Forensics involves capturing, recording, and analyzing network traffic to detect intrusions and investigate security incidents.

Data sources:

  • Full packet capture (PCAP)
  • NetFlow/sFlow/IPFIX
  • DNS query logs
  • Firewall logs
  • IDS/IPS alerts
  • Proxy logs

Analysis tools:

  • Wireshark: Packet analysis
  • Zeek (Bro): Network security monitoring
  • NetworkMiner: Host extraction
  • tcpdump: Command-line capture
  • Moloch/Arkime: Large-scale PCAP

Investigation tasks:

  • Identify data exfiltration
  • Trace attack origin
  • Reconstruct attacker actions
  • Extract transferred files
  • Identify C2 communication
Q26 Explain Zero Trust Network Architecture. Medium +

Zero Trust eliminates implicit trust based on network location. "Never trust, always verify."

Core principles:

  • Verify explicitly: Authenticate and authorize every request
  • Least privilege access: Minimum necessary permissions
  • Assume breach: Design as if attackers are already inside

Key components:

  • Identity verification: Strong authentication, MFA
  • Device validation: Health checks, compliance
  • Microsegmentation: Application-level isolation
  • Continuous monitoring: Real-time risk assessment
  • Data protection: Encryption, DLP

Implementation approaches:

  • Software-Defined Perimeter (SDP)
  • Identity-Aware Proxy
  • Network Access Control
  • Cloud Access Security Brokers (CASB)

Vendors: Zscaler, Cloudflare Access, Google BeyondCorp, Microsoft Zero Trust

Q27 What is traffic tunneling and how is it detected? Hard +

Traffic Tunneling encapsulates one protocol within another to bypass security controls or exfiltrate data.

Common tunneling methods:

  • DNS tunneling: Data in DNS queries/responses (iodine, dnscat2)
  • ICMP tunneling: Data in ping packets (ptunnel)
  • HTTP tunneling: Data in HTTP requests (HTTPTunnel)
  • SSH tunneling: Port forwarding through SSH
  • WebSocket tunneling: Bidirectional over port 80/443

Detection indicators:

  • DNS: High query volume, long TXT records, unusual domain patterns, high entropy
  • ICMP: Large payload sizes, high frequency
  • HTTP: Non-standard headers, encoded data

Detection tools:

  • Zeek scripts for DNS analysis
  • Machine learning anomaly detection
  • Payload inspection
  • Statistical analysis of traffic patterns
Q28 How do you secure network management access? Easy +

Network Management Security Best Practices:

  • Use encrypted protocols: SSH instead of Telnet, HTTPS not HTTP, SNMPv3
  • Dedicated management network: Out-of-band management VLAN
  • Access control lists: Restrict management access by source IP
  • Strong authentication: Complex passwords, MFA, RADIUS/TACACS+
  • Role-based access: Limit privileges by job function
  • Session timeouts: Automatic logout after inactivity
  • Logging: Log all management access, centralize logs
  • Disable unused services: Turn off Telnet, HTTP, SNMPv1/2

Example Cisco configuration:

line vty 0 4
  transport input ssh
  access-class 10 in
  exec-timeout 5 0
!
access-list 10 permit 10.0.0.0 0.0.0.255
Q29 What is network segmentation bypass and how do you test for it? Hard +

Segmentation bypass occurs when traffic reaches segments it shouldn't, defeating the purpose of network isolation.

Common bypass methods:

  • Dual-homed hosts: Systems with NICs in multiple VLANs
  • Misconfigured firewalls: Overly permissive rules
  • VLAN hopping: Switch spoofing, double tagging
  • Routing misconfigurations: Routes between segments
  • Application-layer tunneling: Tunnel through allowed ports
  • Compromised shared services: DNS, AD bridging segments

Testing approach:

  1. Map all network segments
  2. Test connectivity between segments
  3. Look for dual-homed systems
  4. Review firewall rules
  5. Test tunneling possibilities
  6. Check shared service access

Tools: nmap, hping3, traceroute, network diagrams review
